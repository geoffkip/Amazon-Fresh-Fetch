# Preparing a GitHub Release

## Quick Steps

### 1. Clean Up
```bash
# Remove build artifacts
rm -rf dist/ build/

# Remove test files (already done)
# Remove user data (already in .gitignore)
```

### 2. Build Installers

**On macOS:**
```bash
python3 build_standalone.py
# Creates: dist/AmazonFreshFetch-Installer

# Optional: Create DMG
python3 build_mac.py
# Creates: dist/AmazonFreshFetch-Installer.dmg
```

**On Windows:**
```bash
python build_standalone.py
# Creates: dist/AmazonFreshFetch-Installer.exe
```

### 3. Test Installers

- Test on clean system (or VM)
- Verify installation works
- Test app launches correctly

### 4. Create GitHub Release

1. Go to GitHub repository
2. Click "Releases" → "Draft a new release"
3. Tag: `v1.0.0` (or next version)
4. Title: `Version 1.0.0` (or your version)
5. Description: Copy from `RELEASE_NOTES.md`
6. Upload files:
   - `dist/AmazonFreshFetch-Installer.exe` (Windows)
   - `dist/AmazonFreshFetch-Installer.dmg` (macOS) or executable
7. Check "Set as the latest release"
8. Publish release

### 5. Update Documentation

- Update version numbers in code if needed
- Update RELEASE_NOTES.md for next version
- Commit and push changes

## Files to Include in Release

✅ **Include:**
- `AmazonFreshFetch-Installer.exe` (Windows)
- `AmazonFreshFetch-Installer.dmg` or executable (macOS)
- Source code ZIP (auto-generated by GitHub)

❌ **Don't Include:**
- Build artifacts (`dist/`, `build/`)
- User data (`agent_data.db`, `user_session/`)
- Environment files (`.env`)
- Virtual environments (`.venv/`)

## Release Checklist

- [ ] Code is clean and tested
- [ ] Build installers for both platforms
- [ ] Test installers on clean systems
- [ ] Update RELEASE_NOTES.md
- [ ] Update version numbers (if needed)
- [ ] Create GitHub release
- [ ] Upload installers
- [ ] Write release description
- [ ] Publish release
- [ ] Announce release (optional)

## Version Numbering

Follow [Semantic Versioning](https://semver.org/):
- `MAJOR.MINOR.PATCH` (e.g., `1.0.0`)
- MAJOR: Breaking changes
- MINOR: New features, backward compatible
- PATCH: Bug fixes

## Next Steps After Release

1. Share release link with users
2. Update main README if needed
